## Generated artifacts

### `scripts/generate-style-guide.py`

A self-contained CLI script (Python stdlib only) that
analyzes a git repo's commit history and outputs a
complete `style-guide-reference.md`. Ported from
`~/repos/claude_wks/analyze_commits.py` with the
following refactors:

- Built-in `git log` export via `subprocess.run()`
  (eliminates manual `/tmp/*.txt` export step)
- CLI args: `<repo-path> [-n N] [-o path] [--author]`
  (replaces hardcoded `/tmp/modden_commits.txt` and
  `/tmp/tractor_commits.txt`)
- Single-repo-per-run design (original was hardcoded
  to analyze modden + tractor together)
- New `render_style_guide()` function that produces
  a complete markdown style guide matching the format
  of existing per-repo guides in
  `skills/commit-msg/references/`
- New `find_example_commits()` that picks 3
  representative commits (simple/no-body, bullets,
  section markers) for the examples section
- New `get_repo_name()` that extracts name from
  `git remote get-url origin` with fallback to dir
  basename
- New `get_date_range()` that extracts earliest and
  latest commit dates from parsed data

All 6 original analysis functions ported as-is:
`parse_commits()`, `analyze_subjects()`,
`analyze_backticks()`, `analyze_bodies()`,
`analyze_language()`, `analyze_special_patterns()`.
