# NLNet Generative AI Policy — Summary

Source: https://nlnet.nl/foundation/policies/generativeAI/
Version 1.1, effective 2025-12-08.

This reference distills the five key areas of the
NLNet policy so the agent has quick context without
web fetching.

---

## 1. Prompt Provenance Log

**Mandatory** for all substantive AI use. Must
include:
- Model name used
- Dates and times of prompts
- The prompts themselves
- Unedited output from the model

Storage must be logical and persistent — avoid
third-party login-required platforms. In-repo
files under version control are ideal.

## 2. Substantive-Use Marking

Generated content should be marked as such.
For code generation:
- Specify model name/version and how it was used
- Provide prompts and resulting output or summaries
- Include in git commit messages or equivalent
  tracking

**Reduced logging for non-code uses**: testing or
documentation changes need only a general README
description. Per-commit logging preferred but not
strictly required.

## 3. Quality Maintenance

GenAI use must not reduce the quality, clarity,
reliability, or reproducibility of the work.
Humans must understand and explain design/code
decisions.

Outcomes purely generated by AI are not allowed to
be submitted as work eligible for payment without
substantial human contribution.

## 4. No False Attribution

Cannot present AI content as human-authored work.
Grantees should not deliver GenAI outcomes and
pretend it was their own human effort. Human
contributors remain accountable for accuracy and
originality.

## 5. Scope Limitation

GenAI research is permitted only if explicitly
part of the approved project plan. The policy
addresses generative AI specifically (LLMs, code
assistants) — it does not regulate deterministic
code generation, formal proofs, machine learning,
or fuzz testing.

---

## Non-compliance

Failure to comply may result in rejection of the
proposal or termination of the running grant.

## Alternative methods

Grantees may propose equivalent transparency
methods using common-sense judgment, with NLNet
approval.
